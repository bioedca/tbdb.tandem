## [Unreleased]

### Added
- **Similarity-map viewer** (`PhyloTree` on `/tree`) вЂ” the real sequence-derived Stem-I tree, rendered with phylotree.js as an **unrooted** equal-angle layout (the stored Newick is midpoint-rooted for stable layout only; no root or polarity is shown or implied). Built element-level (keyed by `unique_name`) and **collapsed to a per-locus default view**: a locus whose element tips form one clade (sisters вЂ” a duplication) folds to a single locus tip, while loci whose copies are dispersed stay as separate tips, so the **locus в‡„ element toggle** reveals which. A **main Stem-I в‡„ antiterminator-fallback** tree toggle, a **support-collapse slider** (FastTree SH-like supports; weakly-supported branches fade, default < 0.5), and a **non-Firmicutes-only** filter for the 16 outliers. Tips are coloured by specifier (the primary data axis) with a thin neutral phylum-context ring; hovering a tip shows its locus/element, clicking opens its detail page. phylotree.js (legacy d3 v7 + lodash) is dynamically imported so it stays off the boot path. Verified tip counts вЂ” 847 main / 102 fallback elements; 782 / 102 collapsed loci.
- **Tree-load error surface** вЂ” the lazy four-artifact tree load now tracks a status and degrades gracefully (the `/tree` view shows a fallback message and a later visit can retry) instead of leaving an unhandled rejection.
- **Persistent no-polarity banner on `/tree`** вЂ” the standing `NoPolarityBanner` guardrail now sits above the similarity-map viewer with the exact lab-standard text (*"Exploratory similarity map вЂ” unrooted; branch positions reflect sequence similarity, not ancestry."*), pinned as a CI invariant.
- **The tree joins the cross-filtered dashboard** вЂ” a similarity-tree panel now sits on the dashboard and shares the one filter state: it **responds** to the shared selection (loci outside the cross-filter dim, so selecting the ILEГ—LEU cluster narrows the tree to those 10 loci live alongside the table, operon, and heatmap) and **selects** by clicking a tip (narrowing by that locus's specifier, within the existing facet model). The standalone `/tree` route keeps its tip-click-opens-detail behaviour.

The grid renders from the cross-filtered selection so it narrows live, and clicking a cell cross-filters the shared store (setting the specifier + phylum facets, or specifier-only for the unassigned row); Plotly is dynamically imported so it stays off the boot path.

### Fixed
- **Plotly resize-listener leak** вЂ” the dashboard charts no longer use Plotly's `responsive: true` (whose window `resize` listener survives `Plotly.purge`, leaking one per panel on every dashboard mount/unmount); each panel is refitted by a self-cleaning window-`resize` handler that is removed on unmount, so a route round-trip nets zero leaked listeners.
- **Architecture diagram out-of-range feature windows** вЂ” the tandem-architecture diagram now drops a feature window whose end runs past the leader's 3вЂІ end (matching the build's slice guard and the sequence highlighter) instead of clamping it to the body edge, so a corrupt window (e.g. one locus's terminator annotation) no longer draws a phantom glyph; verified to change exactly one element across all 949 members.

## [0.1.0] - 2026-06-02

### Added
- **Data pipeline** (`data-pipeline/build_json.py`) вЂ” deterministic offline build turning the read-only TBDB sources into the committed static artifacts: **470 loci В· 949 canonical members В· 488 intra-locus identity pairs**. Member resolution with the 60-bp core collapse, transcript-5вЂІ ordinals, per-column WUSSв†’dot-bracket structure conversion, the two-tier (EC в†’ regex) `func_class` classifier, the Stem-I length-gate, and `tree_input.fasta` emission. Locked behind 10 validation gates + golden tests.
- **Sequence-derived similarity tree** вЂ” built on the lab cluster (full-leader `cmalign` в†’ RF00230 в†’ Stem-I column slice в†’ FastTree, midpoint-rooted for layout only): `tree.nwk`, `tree_fallback.nwk`, `tree_tips.json`, `tree_locus_map.json` (**847 main-tree tips / 102 fallback**). An exploratory similarity map, not an ancestral-state reconstruction.
- **App shell** вЂ” Vite + Svelte 5 (runes) + Tailwind v4 + TypeScript SPA with hash routing across five routes (Dashboard, Browse, LocusDetail, Tree, About).
- **Design system** вЂ” branded identity: `@theme` + `tokens.ts` design tokens, a 20-AA colorblind-aware specifier palette kept provably disjoint from the chrome palette, self-hosted Inter + JetBrains Mono, and base components (Kpi, FacetChip, Badge, Spinner, TbdbLink, NoPolarityBanner, Card, Button).
- **Data layer + cross-filter store** вЂ” typed staged loading (`loci`/`summary` on boot, `members` in parallel, `identity`/tree lazy) and a single in-memory `$state` filter store with `$derived` selection across every facet (no crossfilter.js).
- **Faceted table** (Tabulator 6) вЂ” all 470 loci with sort, multi-facet filter, free-text search, and CSV export, plus the dashboard KPI strip.
- **Specificity views** вЂ” locus-level specifier-AA bar chart + a symmetric element-pair matrix (preserving the ILEГ—LEU branched-chain cluster) with click cross-filtering, and a bare per-locus detail page with tbdb.io / NCBI deep-links.
- **Tests** вЂ” the Python build-gate + artifact-integrity suite (`pytest`) and the Phase-1 frontend unit + component suite (Vitest + Testing Library).
- **CI/CD** вЂ” GitHub Actions CI (`data` + `web` jobs) and a Pages deploy workflow, with Dependabot and CodeRabbit configuration.
