## [Unreleased]

### Fixed
- **Help (в“ҳ) tooltips now actually open.** The `InfoTip` affordances added across the clarity pass carried their definition only on the native `title` attribute, which never fires on keyboard focus or touch and shows nothing on click вҖ” so every в“ҳ next to a KPI label, chart title, control, or table heading was effectively dead for keyboard and mobile users. `InfoTip` now renders a real, styled popover (the term headword in bold over its definition, on the dark chrome surface so it never reads as a data swatch) shown on hover **and** keyboard focus **and** tap (a tap focuses the button). It is `position: fixed`, so it escapes any `overflow:hidden` card/chart ancestor and renders in full, flips above the trigger when there is no room below, and clamps to the viewport on both axes (so it is never cut off on short/mobile viewports). Escape dismisses it without moving focus (WAI-ARIA tooltip pattern); the popover is `aria-hidden` because the button's `aria-label` already carries the term + definition, so screen readers are not double-read. Covered by a new `InfoTip` component test (hover/focus/Escape, no popover until asked).
- **Final-review accuracy corrections** to the clarity copy above: the `intra-locus pair` definition said identity is scored on **Stem I** вҖ” it is actually scored on the **full leader** (Stem I is what the *similarity map* compares); corrected, and reconciled with the `mean %id` definition. The function-class glossary/legend no longer overclaims "amino-acid biosynthesis/transporter" (the classifier buckets by EC class / keyword, which does not establish substrate scope) вҖ” now "biosynthetic enzyme (transferase/lyase)" and "membrane transporter/permease". The element-comparison subtitle no longer applies a "more negative" sign reading to **О”О”G** (kept as a magnitude, per its definition); only the terminator energy carries the directional reading. The heatmap caption uses the canonical verb ("its elements **sense** different amino acids", not "read").

### Added
- **Reproduce-the-database script + an explicit build-method section on the About page** вҖ” the About page now documents the full `Master_tboxes.csv` вҶ’ loci pipeline step by step (drop contamination вҶ’ orient each T-box on the genome вҶ’ 600-bp single-linkage clustering вҶ’ 60-bp collapse to physical cores вҶ’ the share-a-gene / share-a-specifier / overlapping-leaders tandem test вҶ’ order 5вҖІвҶ’3вҖІ and derive every field), replacing the thinner detection blurb. Alongside it, a new **"Reproduce it yourself"** card offers a self-contained, downloadable Python script вҖ” `reproduce_tandem_tbox_db.py`, committed under `public/` and served from the Pages base path вҖ” that regenerates the whole dataset from the one public source file. Given `Master_tboxes.csv` it rebuilds `loci/members/identity/summary.json`, the tree-input FASTAs, and an optional readable `tandem_loci.tsv`, then self-verifies on exit (470 / 949 / 488, balanced structures, the CP045927 golden). Reproduction is faithful to the published dataset: the counts, the 394/76 confidence split, and the specifier/phylum distributions match exactly, and **943 of 949 element records are byte-identical**; the few differences (physical cores annotated by several redundant source files, where the published table made a source-specific pick; locus IDs re-assigned in deterministic genomic order) are documented in the script header and the About card. The script's detection stage is a **verified reconstruction** of the upstream locus-detection algorithm, which has no surviving source on disk. Covered by updated About component + e2e tests (new download-link assertions).
- **First-contact primer + shared glossary** вҖ” the Dashboard now opens with a one-paragraph, plain-language definition of what a T-box riboswitch and a tandem locus are (a T-box binds one specific tRNA, senses whether it is charged with its amino acid, and switches the downstream gene **on** when that amino acid is scarce; a tandem locus stacks two or more elements in one leader), plus a "New here? вҶ’ About & method" link. A single-source glossary (`glossary.ts`) defines the load-bearing vocabulary once вҖ” present-tense and structural, no evolutionary direction вҖ” and is surfaced inline through a small, keyboard-reachable в“ҳ affordance (`InfoTip`) so a newcomer can resolve a term in place. The About page gains a "what a T-box is" lead paragraph and plain-language glosses of the detection pipeline (the `core5` 5вҖІ anchor; single-linkage clustering as "chaining nearby cores").

### Changed
- **Renamed the project to `tbdb.tandem`** (was *TandemView* / `tandem-tbox-explorer`) вҖ” the app, the GitHub repo, and the GitHub-Pages URL now read as a sub-brand of [tbdb.io](https://tbdb.io). The site moves to **https://bioedca.github.io/tbdb.tandem/**; the old `вҖҰ/tandem-tbox-explorer/` Pages URL stops resolving (GitHub redirects the repo, release, and git URLs, but not the Pages sub-path). The Vite build base, the in-app wordmark/title/footer/loading copy, the favicon label, the CSV export name (`tbdb-tandem-loci.csv`), the reproduce-script header, and the docs were updated in lockstep; the frontend's data fetches already resolve through the Vite base (`import.meta.env.BASE_URL`), so no data paths changed.
- **KPI strip relabelled for first-time readers** вҖ” the tiles that conflated or mislabelled their numbers are fixed: "Triples / 461 pairs" (which paired a headline of 9 three-element loci with a sublabel about 461 two-element loci) now reads "Triples 9 В· 3-element loci В· 461 are pairs"; "Non-Firmicutes / outlier loci" вҶ’ "loci outside Firmicutes" (a taxonomic fact, not a statistical/quality judgment); "Members / canonical cores" вҶ’ "individual T-box cores"; "Intra-locus pairs / %-identity pairs" вҶ’ "element pairs within a locus"; and every tile now carries an в“ҳ definition. The nav item "About" вҶ’ "About & method" (matching the page's own heading), and the About count tile "canonical members" вҶ’ "T-box elements".
- **Locus-detail clarity** вҖ” the signature **architecture diagram now carries a glyph legend** (T-box element, Stem I, Stem-I loop, specifier codon, antiterminator, terminator, discriminator, downstream gene/operon, transcription direction, spacer), and its per-element tags are plain numbers (1вҖҰn in 5вҖІвҶ’3вҖІ order) so they no longer collide with the global 5вҖІ/3вҖІ leader-end caps; the translational `SD?` glyph is relabelled **`SD/RBS`** with an explanatory tooltip. The element-comparison table defines `О”О”G` and renames `Term E` вҶ’ **`Term О”G`**, both with units (kcal/mol) and в“ҳ tooltips вҖ” О”О”G is described generally (a free-energy difference between the two folds) rather than asserting an undocumented sign convention вҖ” and the Specifier column, the Pairwise-identity heading, and the Mean %id field gain в“ҳ definitions (including the shared-leader 100% caveat). The locus header chips (same/mixed, regulation type, cores) get hover tooltips, and the RNA panel now reads "in-app preview (approximate layout)" with the antiterminator fold (the tRNA-bound, gene-ON state) and VARNA defined.
- **Similarity-map clarity** вҖ” the nav item "Tree" вҶ’ "Similarity map" (it is a sequence-similarity map, not a phylogeny), and the page intro now says what Stem I is and *why* the map is unrooted (no time axis, no outgroup вҶ’ a cluster means the sequences resemble each other, not a lineage or order of appearance). Every control gains an inline definition (в“ҳ): the View (locus vs element) toggle, the Tree toggle (Main В· Stem-I vs the relabelled **"Fallback (antiterminator)"**), the branch-support slider, and the non-Firmicutes filter. The legend defines "specifier", labels the grey **"? unassigned"** dot, and expands the phylum outer-ring note; the caption adds a how-to-read hint for the long branches. "outliers" вҶ’ "minority" throughout.
- **Chart legends & captions on the dashboard panels** вҖ” the previously key-less charts now explain themselves. The specificity overview defines "specifier" inline (в“ҳ) and adds a legend for the element-pair matrix (each cell = two-element loci by their two specifiers; **diagonal** = same-specifier, **off-diagonal** = mixed; teal = count; `?` = unresolved; folded/symmetric) and the triple-core list (each dot = an element's specifier in 5вҖІвҶ’3вҖІ order); the ILEГ—LEU aside is restated precisely as the largest pairing of two *resolved* specifiers. The specifierГ—phylum heatmap defines specifier, flags mixed (`A;B`) loci, and drops the loaded word "outliers". The regulated-operon panel states the riboswitchвҶ’operon link, spells out the function classes (aaRS = aminoacyl-tRNA synthetase, biosynthesis, transporter, oxidoreductase, unknown) and the classification source (EC number / text-inferred\* / none), and clarifies the Sankey bands (loci count of a specifier regulating a function class вҖ” an observed association, not a cause).

## [0.3.0] - 2026-06-03

### Added
- **Similarity-map viewer** (`PhyloTree` on `/tree`) вҖ” the real sequence-derived Stem-I tree, rendered with phylotree.js as an **unrooted** equal-angle layout (the stored Newick is midpoint-rooted for stable layout only; no root or polarity is shown or implied). Built element-level (keyed by `unique_name`) and **collapsed to a per-locus default view**: a locus whose element tips form one clade (sisters вҖ” a duplication) folds to a single locus tip, while loci whose copies are dispersed stay as separate tips, so the **locus вҮ„ element toggle** reveals which. A **main Stem-I вҮ„ antiterminator-fallback** tree toggle, a **support-collapse slider** (FastTree SH-like supports; weakly-supported branches fade, default < 0.5), and a **non-Firmicutes-only** filter for the 16 outliers. Tips are coloured by specifier (the primary data axis) with a thin neutral phylum-context ring; hovering a tip shows its locus/element, clicking opens its detail page. phylotree.js (legacy d3 v7 + lodash) is dynamically imported so it stays off the boot path. Verified tip counts вҖ” 847 main / 102 fallback elements; 782 / 102 collapsed loci.
- **Tree-load error surface** вҖ” the lazy four-artifact tree load now tracks a status and degrades gracefully (the `/tree` view shows a fallback message and a later visit can retry) instead of leaving an unhandled rejection.
- **Persistent no-polarity banner on `/tree`** вҖ” the standing `NoPolarityBanner` guardrail now sits above the similarity-map viewer with the exact lab-standard text (*"Exploratory similarity map вҖ” unrooted; branch positions reflect sequence similarity, not ancestry."*), pinned as a CI invariant.
- **The tree joins the cross-filtered dashboard** вҖ” a similarity-tree panel now sits on the dashboard and shares the one filter state: it **responds** to the shared selection (loci outside the cross-filter dim, so selecting the ILEГ—LEU cluster narrows the tree to those 10 loci live alongside the table, operon, and heatmap) and **selects** by clicking a tip (narrowing by that locus's specifier, within the existing facet model). The standalone `/tree` route keeps its tip-click-opens-detail behaviour.
- **About / method page** (`/about`) вҖ” the provenance and caveats note: how the 470 tandem loci are detected (the upstream contamination-drop вҶ’ strand/`core5` projection вҶ’ 600-bp single-linkage вҶ’ 60-bp collapse вҶ’ вүҘ2-distinct-cores pipeline), the data caveats that govern every view (the contamination drop, the corrupt raw-codon column so specifiers come from `amino_acid_top`/`refine_codon_top`, low-confidence loci kept-and-flagged, the near-monochrome taxonomy), the **no-polarity disclaimer** (the similarity map is an exploratory map, not a phylogeny or ancestral-state reconstruction вҖ” with the standing banner re-shown and the tree method summarised), the TBDB provenance + citation (CC-BY; Marchand et&nbsp;al. 2021, with the DOI and citing-page links), and a glossary. The load-bearing counts (470 / 949 / 488, confidence, non-Firmicutes) are bound live from `summary.json` rather than hardcoded.
- **End-to-end + visual-regression suite** (Playwright, pinned) вҖ” runs against the production build served by `vite preview` at the real Pages base path, across all five routes: the **Dashboard** (KPI values match `summary.json`; selecting the ILEГ—LEU matrix cell narrows the table, tree, and Sankey to the 10 branched-chain loci; clicking a dashboard tree tip narrows the shared store by specifier), **Browse** (search, facet filter, column sort, CSV export), **LocusDetail** (architecture diagram + element comparison + an RNA tab, with the tbdb.io and NCBI deep-links resolving), **Tree** (renders with the mainвҶ”fallback toggle switching trees, the no-polarity banner present), and **About** (the method-note sections and attribution links). Pixel baselines cover the deterministic surfaces only вҖ” the dashboard chrome/layout (Plotly + tree masked), the architecture diagram, the similarity tree (fixed seed + layout), and the KPI strip вҖ” with the force-directed fornac RNA excluded and a small `maxDiffPixelRatio` absorbing sub-pixel noise. CI's `web` job now installs the pinned browser and runs the suite after the build, uploading the report + snapshots on failure.

## [0.2.0] - 2026-06-03

### Added
- **Tandem-architecture diagram** (`ArchitectureDiagram`) вҖ” the signature view: a to-scale, biological-5вҖІвҶ’3вҖІ locus track rendering per element the T-box body (tinted by its own specifier, so mixed loci read two-tone), Stem-I with a notched terminal loop, the specifier-codon tick + amino-acid code, the antiterminator outline, the terminator hairpin (a schematic anti-SD sequestrator for translational elements), and the discriminator вҖ” plus dashed inter-element spacers labelled with their bp gap, strand chevrons, a scale bar, and a function-class-tagged downstream-ORF block arrow. Hand-rolled SVG with D3 supplying only the linear position scale.
- **Element-comparison panel** (`ElementComparison`) вҖ” per-element specifier/codon, tRNA family, О”О”G and terminator-energy as in-cell bars, completeness dots, and tbdb.io / NCBI deep-links, with the intra-locus pairwise %-identity (from `identity.json`); collapse-recovered shared-leader loci are flagged where their leader identity saturates at 100%.
- **Full per-locus detail page** (`/locus/:id`) вҖ” wires the complete detail flow from the in-memory members map with no per-locus network call: the tandem-architecture diagram, the element-comparison panel, and **feature-highlighted member sequences** (`MemberSequence`) that render each element's gap-free leader with its Stem-I, specifier-codon, antiterminator, terminator, and discriminator spans highlighted (specifier-tinted fill for the Stem-I/codon, neutral chrome rules for the regulatory features). Intra-locus identity loads lazily and degrades gracefully when unavailable.
- **In-app RNA secondary structure** (`RnaStructure`) on the detail page вҖ” a best-effort [fornac](https://github.com/pkerpedjiev/fornac) render of each element's whole-leader antiterminator fold, one tab per element, with the tbdb.io VARNA structure view always offered per element as the guaranteed deep-link. The legacy fornac UMD loads lazily as a classic `<script>` (kept off the boot path and out of ESM strict mode); its injected global `svg` styles and its otherwise-permanent window resize listener are both neutralized so they cannot affect the rest of the app. Honors `prefers-reduced-motion`.
- **Specifier Г— phylum heatmap** (`SpecPhylumHeatmap`) on the dashboard вҖ” a locus-level count grid of specifier (Г—27, frequency-descending) by phylum (Г—7, including an `unassigned` row for the 3 loci with no phylum), so all 470 loci are represented and the 16 non-Firmicutes outliers stand out against the near-monochrome Firmicutes mass. Cells use a separate neutral slate ramp (a chrome magnitude scale, never the 20-AA specifier palette), with counts printed in-cell. The grid renders from the cross-filtered selection so it narrows live, and clicking a cell cross-filters the shared store (setting the specifier + phylum facets, or specifier-only for the unassigned row); Plotly is dynamically imported so it stays off the boot path.
- **Regulated-operon breakdown** (`OperonBreakdown`) on the dashboard вҖ” three coordinated pieces over the one shared filter state: в‘  horizontal **stacked bars** of loci by downstream `func_class`, split by the two-tier classifier's provenance (solid = EC-backed В· hatched = text-inferred\* В· dotted = no annotation); в‘Ў a **specifier вҶ’ function-class Sankey** exposing the observed couplings (TRP вҶ’ biosynthesis, THR вҶ’ aaRS, the ILE/LEU branched-chain cluster вҶ’ biosynthesis); and в‘ў the regulation `type` shown as **chips** (no transcriptional/translational toggle). Bars colour by a neutral `func_class` chrome ramp (now shared with the architecture diagram) and the Sankey's specifier nodes use the data palette, preserving the chrome вҹӮ data separation. Counts and flows render from the cross-filtered selection so the whole panel narrows live, and clicking a bar cross-filters the shared store. Every label states an observed association only вҖ” no evolutionary-direction language. Plotly is dynamically imported so it stays off the boot path.
- **Cross-filtered dashboard centerpiece** вҖ” every viz panel and the faceted table now share one filter state, so selecting a value in any chart narrows the whole dashboard live: clicking the ILEГ—LEU matrix cell narrows the table, the operon bars, and the specifierГ—phylum heatmap to those 10 loci. A brief list fade/reflow on the table rows makes the narrow read as live (honoring `prefers-reduced-motion`); the KPI strip stays the fixed dataset overview while the live selection count lives in the table toolbar.
- **WUSSвҶ”build parity + the Phase-2 test suite** вҖ” a frontend `wuss.ts` mirror of the Python build's WUSSвҶ’dot-bracket converter, asserted to parity against the same golden Stem-I structures the build is pinned to, plus component tests for the architecture diagram (glyph set, 5вҖІвҶ’3вҖІ order, two-tone mixed loci, triples, the translational anti-SD variant), the specifierГ—phylum heatmap, the operon stacked bars + Sankey, and the RNA viewer's deep-link fallback, and pure-logic unit tests with real-artifact drift guards for the operon couplings, the phylum census, and the RNA render basis.

### Fixed
- **Plotly resize-listener leak** вҖ” the dashboard charts no longer use Plotly's `responsive: true` (whose window `resize` listener survives `Plotly.purge`, leaking one per panel on every dashboard mount/unmount); each panel is refitted by a self-cleaning window-`resize` handler that is removed on unmount, so a route round-trip nets zero leaked listeners.
- **Architecture diagram out-of-range feature windows** вҖ” the tandem-architecture diagram now drops a feature window whose end runs past the leader's 3вҖІ end (matching the build's slice guard and the sequence highlighter) instead of clamping it to the body edge, so a corrupt window (e.g. one locus's terminator annotation) no longer draws a phantom glyph; verified to change exactly one element across all 949 members.

## [0.1.0] - 2026-06-02

### Added
- **Data pipeline** (`data-pipeline/build_json.py`) вҖ” deterministic offline build turning the read-only TBDB sources into the committed static artifacts: **470 loci В· 949 canonical members В· 488 intra-locus identity pairs**. Member resolution with the 60-bp core collapse, transcript-5вҖІ ordinals, per-column WUSSвҶ’dot-bracket structure conversion, the two-tier (EC вҶ’ regex) `func_class` classifier, the Stem-I length-gate, and `tree_input.fasta` emission. Locked behind 10 validation gates + golden tests.
- **Sequence-derived similarity tree** вҖ” built on the lab cluster (full-leader `cmalign` вҶ’ RF00230 вҶ’ Stem-I column slice вҶ’ FastTree, midpoint-rooted for layout only): `tree.nwk`, `tree_fallback.nwk`, `tree_tips.json`, `tree_locus_map.json` (**847 main-tree tips / 102 fallback**). An exploratory similarity map, not an ancestral-state reconstruction.
- **App shell** вҖ” Vite + Svelte 5 (runes) + Tailwind v4 + TypeScript SPA with hash routing across five routes (Dashboard, Browse, LocusDetail, Tree, About).
- **Design system** вҖ” branded identity: `@theme` + `tokens.ts` design tokens, a 20-AA colorblind-aware specifier palette kept provably disjoint from the chrome palette, self-hosted Inter + JetBrains Mono, and base components (Kpi, FacetChip, Badge, Spinner, TbdbLink, NoPolarityBanner, Card, Button).
- **Data layer + cross-filter store** вҖ” typed staged loading (`loci`/`summary` on boot, `members` in parallel, `identity`/tree lazy) and a single in-memory `$state` filter store with `$derived` selection across every facet (no crossfilter.js).
- **Faceted table** (Tabulator 6) вҖ” all 470 loci with sort, multi-facet filter, free-text search, and CSV export, plus the dashboard KPI strip.
- **Specificity views** вҖ” locus-level specifier-AA bar chart + a symmetric element-pair matrix (preserving the ILEГ—LEU branched-chain cluster) with click cross-filtering, and a bare per-locus detail page with tbdb.io / NCBI deep-links.
- **Tests** вҖ” the Python build-gate + artifact-integrity suite (`pytest`) and the Phase-1 frontend unit + component suite (Vitest + Testing Library).
- **CI/CD** вҖ” GitHub Actions CI (`data` + `web` jobs) and a Pages deploy workflow, with Dependabot and CodeRabbit configuration.
