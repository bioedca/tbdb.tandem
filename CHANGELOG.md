## [Unreleased]

### Added
- **The "Reproduce it yourself" script can now generate the genomic context itself, and its `members.csv` carries the same genomic columns as the published one.** `public/reproduce_tandem_tbox_db.py` mirrors the twelve genomic-context columns (above) — blank by default, since `Master_tboxes.csv` has no genomic coordinates — and gains an opt-in **`--genomic-context`** mode that efetches each locus's downstream-gene coordinates + interval from NCBI (the *only* network path in the script; cached under `--ncbi-cache`, `--offline` rebuilds from cache, `--email`/`NCBI_EMAIL` required), writes `<out>/locus_context/<id>.json`, and fills those columns; `--locus-context DIR` reuses an existing fetch. The fetch engine is a self-contained port of `data-pipeline/{ncbi_ids,fetch_genomic_context}.py` (lazy `Bio.Entrez`, so the default run still needs no network), guarded by a **faithfulness cross-check** that asserts the ported `build_locus_record` is byte-equal to the maintainer module on `+`/`−` fixtures, plus fill, secret-hygiene (the email/API key never reach a cache path, artifact, or log — fetch failures print the exception *type* only), CLI-validation, and golden-blank tests. The byte-for-byte golden (default, network-free) gains the twelve blank columns; everything else is unchanged.
- **`members.csv` now carries the per-locus genomic context, so the downloadable base table can reconstruct the continuous `/locus` view.** The flat member table gains twelve NCBI-derived columns: `genomic_resolved`, the genomic `locus_interval_start`/`_end`, each element's `element_offset` within that interval, and the compact facts of the locus's **proximal regulated downstream gene** (`downstream_gene_name`/`_id`/`_locus_tag`/`_start`/`_end`/`_strand`/`_offset`) plus a `downstream_gene_count` operon-size flag. These were previously only in the per-locus `public/data/locus_context/<id>.json` files (the `fetch_genomic_context.py` NCBI step) — nothing in the consolidated table pointed back to them. The gene's genomic span is derived from its transcription-oriented offset + length (the inverse of the context build's `offset_5p`), and the "primary" gene is the most-5′ gene **co-oriented with the locus** (a T-box regulates a gene on its own strand) — not the smallest-offset neighbour, which on the 4 mixed-strand loci (T0108/T0241/T0278/T0352) is an opposite-strand gene. The bulky interval `seq` is deliberately left out of the CSV (it stays in the JSON; `locus_interval_*` + `element_offset` place each element within it). `build_json.py` fills the columns from the committed `locus_context/` (blank when absent — a Master-only build carries no genomic context), and the committed `public/data/members.csv` is regenerated. Covered by an independent-oracle test that re-derives every genomic cell straight from `locus_context/<id>.json` (guarding members.csv ⇄ locus_context staleness) and a blank-mode additivity check.
- **3D similarity cloud (`/cloud`) — an interactive WebGL companion to the 2D similarity map.** The same Stem I sequence-similarity relationships, rendered as a 3D point cloud so the dense Firmicutes core stops being an unreadable flat hairball: the patristic-distance structure is genuinely high-dimensional, and a 3D **classical-MDS / PCoA** embedding recovers **~41 %** of the pairwise distance vs **~32 %** for a flat 2D layout (the shorter antiterminator fallback tree: ~84 % / ~75 %) — both percentages are computed from the embedding's eigen-variance and shown in a standing readout. Same no-polarity guardrail as `/tree` (the `NoPolarityBanner`, and patristic distance is rooting-independent so no ancestry is read), and the same locus ⇄ element / main ⇄ fallback toggles. A new build step (`data-pipeline/build_cloud.py`) computes the embedding offline into a committed `public/data/cloud.json` (847 + 102 points, < 250 KB) with a **deterministic sign convention** so the file is byte-stable across rebuilds; the patristic matrix is computed with a small pure-Python Newick walk (no new `dendropy` dependency). The crowded core is de-piled for clickability by an **anchored-repulsion "spread"** slider that repels nearby points while springing each toward its true PCoA anchor — it is fully **reversible** (spread 0 returns *exactly* to the metric positions), monotone, bounded, and neighbourhood-preserving, and it uses a uniform spatial hash (never an O(n²) scan); the UI surfaces the resulting mean offset and reminds the reader it is "inflated for clarity." Points can be coloured + sized by five vetted **presets** (Specifier · Switch strength · Function · QC/confidence · Taxonomy) or a free color×size **Advanced** disclosure, with a nearest-neighbour **constellation** overlay, a **non-Firmicutes** emphasis (by size, not colour alone), reduced-motion-aware idle rotation, hover tooltips, and click-through to the locus detail page (or specifier cross-filter when used as a dashboard panel). The cloud joins the shared cross-filter (out-of-selection loci dim, never a rebuild). `three` is **dynamically imported** and the route is **code-split**, so neither `three` (732 KB) nor `cloud.json` enters the boot bundle. All point logic (the spread stepper, encodings, element↔locus aggregation, orbit-camera math) lives in pure, unit-tested `src/lib/cloud/*` modules, mirroring how `tree.ts` stays framework-agnostic. Covered by new unit (relax invariants, encodings, aggregation, orbit math, cloud.json data-contract), component (mocked-WebGL smoke), e2e, and data-pipeline (PCoA determinism + committed-artifact integrity) tests. Method note: [`docs/similarity-cloud.md`](docs/similarity-cloud.md).

### Fixed
- **Refreshed the stale reproduce-script golden baseline so its byte-for-byte regression suite passes again.** The `data-pipeline/tests/golden/` snapshot (the Phase-0 oracle the `reproduce_tandem_tbox_db.py` tests diff against) predated a later improvement to the **downstream-function classifier** (the EC-number extraction + `classify_func` regexes), which landed in both `build_json.py` and the reproduction script but was never recaptured in the golden. As a result `test_phase1_byte_identical` / `test_phase2_*` were failing locally for anyone with the source CSV (they `@needs_master`-skip in CI, so the breakage was invisible there). The current reproduce script matches the authoritative committed `public/data` **exactly** (0/465 joinable loci disagree on `func_class`/`func_source`), confirming the script is correct and the snapshot was stale; the golden is regenerated under the pinned `biopython==1.87` / `pandas==3.0.3`. Only the classifier-derived files moved — `loci.json`, `members.json`, `summary.json`, `members.csv`, `tandem_loci.tsv` (`downstream.func_class`/`func_source`/`ec` + their rollups) — while `identity.json`, `tree_input.fasta`, and `antiterm_fallback.fasta` are byte-identical to before. No script or app change; test-data only.
- **R2DT gray single strands no longer sprawl as long straight runs that bloat the illustration — long tails fold into a compact serpentine.** A dangling 5′/3′ single-stranded tail was drawn as a straight ray of `k` median steps, so a typical antiterminator leader (a *median of 42 nt* before Stem I, up to 89 nt) shot a long spike across the canvas, pushing the diagram's bounding box wide and empty — at the fit-zoom the whole illustration ended up large with tiny glyphs. The single-strand router (`build_r2dt.py`) now folds a long tail (`k ≥ 8`) into a compact **boustrophedon serpentine**: it seeds the fold along the clearest open wedge the straight ray would have escaped into, tries row counts from a thin 2-row hairpin up to a bushy block, and takes the **most compact placement that still clears the structure** (the straight ray and keep-current stay candidates, so a tail with no clear fold is left as a ray — do no harm). Long, *slacky* interior connectors (anchors close together, R2DT bowed them into a wide arc) likewise fold into a tight sawtooth that hugs the chord; genuinely **taut** runs (anchors far apart, which a fold cannot shrink) are left straight. Across the committed corpus this folds **99 % of antiterminator** and **95 % of terminator** long tails, cutting the total antiterminator diagram bounding-box area by **~32 %**, while glyph overlaps stay flat-to-lower (antiterm 218 → 212, term 455 → 449 — no new clashes). Folding is confined to the **final post-fold reflow** so the stems-only base reflow is byte-identical to before — the antiterminator⇄terminator **stem pin is untouched** (every shared element still pins Stem I/II/III, only degenerate *Partial* leaders exempt) — and it is fully **deterministic** (rounded clearance/extent keys, fixed candidate order), so a shared tail folds identically in both conformations and never jumps on toggle. Paired residues never move; `seq`, `pairs`, and residue indices stay untouched, so the stem/motif colouring stays valid. Committed counts are unchanged (antiterminator **790**, terminator **784**); the committed `public/data/r2dt/*.json` + `r2dt/term/*.json` are regenerated. Covered by synthetic router unit tests (fold compactness/continuity/self-clearance, slacky-vs-taut, do-no-harm, base-reflow-does-not-fold, determinism) and corpus guards (≥ 90 % of long tails folded cleanly, and a whole-corpus visual-overlap ceiling).
- **R2DT internal loops now stay open across the whole corpus — the readability pass no longer cramps a well-separated loop back into one arch.** The coordinated internal-loop open added previously still left the two strands of ~230 loops running ~1 median step apart at the fit-zoom (the very "blur into one arch" it was meant to cure), and on ~59 of those it actively *reduced* a separation R2DT had already drawn wide (e.g. 3.2 → 1.0 steps). Two causes: the side-assignment was chosen to maximise `min(clearance, separation)`, which pins the separation to the ~1-step clearance ceiling instead of opening the strands as far apart as they can go; and when that capped candidate missed the strict `R2DT_LOOP_STRAND_SEP_RATIO` gate, the loop fell back to opening each strand **independently**, which could bow the 5′ and 3′ strands toward each other. The pass now treats a multi-residue internal loop (both strands ≥ 2 nt) as a single decision: each strand may keep its drawn position or bow to either side that clears the rest of the structure, and the pass places the option-pair that **maximises** the 5′↔3′ separation while **never** dropping below the gap the committed layout already has (strict do-no-harm) — so loops R2DT drew wide stay wide, cramped loops open as far as clearance allows, and a loop with no clear opening is left exactly as drawn instead of being crushed. Single-residue bulges still use the one-sided opener (they cannot read as a continuous loop and need the off-axis bow to be seen). Across every committed `r2dt/` and `r2dt/term/` diagram, every multi-residue internal loop now comes out **no tighter than R2DT drew it** (well-separated coverage 98.6 % → 99.8 %, the remainder being loops R2DT itself drew cramped with no room to open). Paired residues, `seq`, `pairs`, and residue indices stay untouched. Covered by a new corpus-wide do-no-harm separation guard over every committed diagram and a max-separation-not-balanced-min unit test, alongside the existing opposite-sides, T0185, and single-residue guards.
- **R2DT internal loops no longer run together in one arch — each internal loop's two strands now open to opposite sides.** A Stem-I internal loop has unpaired residues on *both* strands of the helix, but the previous readability pass opened every unpaired run independently, so the 5′ and 3′ strands of one loop could bow to the same side and sit a fraction of a step apart — at the fit-zoom, where the glyphs draw at full size, they blurred into a single arch that read as one continuous loop even though the residues across it are not base-paired. The display-only loop pass now recognises an internal loop (a 5′ unpaired run with a matching 3′ unpaired run between the partners of its two closing pairs) and opens it as a **coordinated pair**: the two strands bow to opposite sides of the helix axis, separated by at least `R2DT_LOOP_STRAND_SEP_RATIO` (1.15) median steps so the opening reads as two distinct strands. Bulges, hairpin/terminal loops, and single-residue loops keep the original one-sided outward bow, and an internal loop whose closing pairs are packed too tightly to separate falls back to that same bow — so the pass is strictly do-no-harm: it never buries two glyphs deeper than the spaced input already packs them. Paired stem residues, `seq`, `pairs`, and the residue indices stay untouched, so the stem/motif colouring and the antiterminator⇄terminator stem pinning remain valid, and it applies to **all** R2DT illustrations (both conformations, every visible stem and the terminator hairpin), leveraging the room the Stem I→Stem II spacer already opens up.
